# 自我更新机制（活体 skill 的引擎）

本技能随服务器持续抓取的 Serenity 最新帖子自我迭代。管线在新加坡服务器 /opt/xwatch 下。

## 数据分层
- **稳定层（慢变量）**：SKILL.md、mental-models.md、heuristics.md、voice-dna.md、method-engine.md。改动需用户飞书确认。
- **快照层（快变量）**：market-pulse.md、universe.md、credibility-map.md、claims/。由管线自动刷新+留痕。

## 预测-验证-纠偏闭环（核心）
1. **沉淀**：每条她的新帖 → 提取"主张卡"（标的+方向+逻辑+隐含可验证预测+到期复查时间+当时锚点价/市值）。存飞书"主张验证"表。
2. **补全 why**：用心智模型反推她没明说的隐含逻辑（标注框架推断）。
3. **到期联网验证**：到复查时间，联网查真实结果（股价/市值/催化剂/财报/第三方观点），给主张打分：应验/部分应验/证伪/待观察 + 依据+来源。
4. **纠偏**：累积战绩 → 更新 credibility-map.md（她哪类判断历史命中率高/低）。高命中逻辑加权，常翻车模式标警惕。
5. **预测+自评**：基于修正模型预判她下一步 → 后续用真实结果给"我们的预判"打分 → 算"懂她指数"。

## 蒸馏节奏
- 每周一次 + 攒够 30 条她的新帖提前触发。
- 产出"懂她复盘"卡片到飞书：她近期对错 + 我们的命中率(懂她指数) + 模型调整建议。
- 快照层自动更新；稳定层改动 → 飞书确认卡片，采纳后写入并记版本。

## 版本与回滚
- evolution/changelog.md 记录每次迭代（改了什么、依据哪些帖子/验证）。可回滚。

## 免责
所有预测/补全 why 统一标注：AI 框架推断、非她本人、可能错、非投资建议。
